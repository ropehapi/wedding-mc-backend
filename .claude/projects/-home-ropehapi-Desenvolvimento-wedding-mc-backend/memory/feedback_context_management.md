---
name: Context window management
description: When context window approaches 40%, save progress to memory, clear context, and resume
type: feedback
---

When the context window approaches 40% usage, save the current progress state to a memory file, clear the context window (/clear), and resume the implementation task from where it stopped.

**Why:** User preference to keep context lean and avoid context overflow during long implementation sessions.

**How to apply:** Monitor context usage during long multi-file implementation tasks. At ~40%, persist progress notes and resume.
